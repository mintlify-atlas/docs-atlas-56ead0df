## Calendar Overview

The calendar system helps you manage your availability and appointments. Clients can view your calendar when booking services, and you can track upcoming jobs.

## Accessing the Calendar

Access your calendar from your profile:

```tsx
import { Calendar } from 'lucide-react';
import Link from 'next/link';

const handleClickCalendar = () => {
  sessionStorage.setItem('fixer_id', fixer.id);
  sessionStorage.setItem('requester_id', 'pupup');
  sessionStorage.setItem('roluser', 'fixer');
};

<Link
  href='../calendar'
  onClick={handleClickCalendar}
  className='px-4 py-2 bg-white text-blue-700 rounded-lg flex items-center gap-2'
>
  <Calendar className='w-4 h-4' />
  Ver Calendario
</Link>
```

## Role-Based Calendar Access

The calendar adapts based on user role:

```typescript
interface CalendarSession {
  fixer_id: string;      // The fixer whose calendar is being viewed
  requester_id: string;  // The user viewing the calendar
  roluser: 'fixer' | 'requester';  // Current user's role
}
```

### Fixer View

When a fixer views their own calendar:
- Can add/edit/delete availability blocks
- See all booked appointments
- Manage time slots
- Block out unavailable times

### Requester View

When a client views a fixer's calendar:
- Can only see available time slots
- Can request appointments
- Cannot modify fixer's schedule

## Calendar Features

### Availability Management

Fixers can define their available hours:

```typescript
interface AvailabilitySlot {
  dayOfWeek: number;  // 0-6 (Sunday-Saturday)
  startTime: string;  // HH:mm format
  endTime: string;    // HH:mm format
  isAvailable: boolean;
}
```

### Appointment Tracking

Track scheduled appointments:

```typescript
interface Appointment {
  id: string;
  fixerId: string;
  requesterId: string;
  serviceType: string;
  scheduledDate: Date;
  startTime: string;
  endTime: string;
  status: 'pending' | 'confirmed' | 'completed' | 'cancelled';
  notes?: string;
}
```

### Blocked Times

Mark times when you're not available:

```typescript
interface BlockedTime {
  id: string;
  fixerId: string;
  startDateTime: Date;
  endDateTime: Date;
  reason?: string;
}
```
