## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 20.x or higher
- **npm** 11.x or higher (comes with Node.js)
- **Git** for cloning the repository

## Quick Setup

<Steps>
  <Step title="Clone the repository">
    Clone the Servineo frontend repository to your local machine:

    ```bash
    git clone <repository-url>
    cd servineo-frontend
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using npm:

    ```bash
    npm install
    ```

    This will install key dependencies including:
    - **Next.js 15.5.4** - React framework with App Router
    - **React 19.1.0** - UI library
    - **Redux Toolkit 2.10.1** - State management
    - **TypeScript 5.9.3** - Type safety
    - **Tailwind CSS 4.1.17** - Styling
  </Step>

  <Step title="Configure environment variables">
    Create a `.env.local` file in the root directory with the following variables:

    ```bash .env.local
    # API Configuration
    NEXT_PUBLIC_API_URL=http://localhost:3000
    NEXT_PUBLIC_BACKEND=http://127.0.0.1:8000
    NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000/api

    # Optional: Google OAuth (for social login)
    NEXT_PUBLIC_GOOGLE_CLIENT_ID=your_google_client_id

    # Optional: Google reCAPTCHA
    NEXT_PUBLIC_RECAPTCHA_SITE_KEY=your_recaptcha_site_key
    ```

    <Warning>
      The backend API is configured to run on port 8000. Make sure your backend service is running or update the URLs to match your backend configuration.
    </Warning>
  </Step>

  <Step title="Start the development server">
    Run the development server with Turbopack for faster builds:

    ```bash
    npm run dev
    ```

    The application will start at [http://localhost:3000](http://localhost:3000)

    <Note>
      Servineo uses Next.js with Turbopack enabled for improved development performance. The server runs on port 3000 by default.
    </Note>
  </Step>

  <Step title="Explore the platform">
    Open your browser and navigate to `http://localhost:3000`. You'll see the Servineo home page with:

    - **Hero Section** - Platform introduction
    - **Services Section** - Browse available service categories
    - **Map Section** - Find nearby fixers
    - **How It Works** - Platform explanation

    The home page is located at `src/app/[locale]/page.tsx:23`
  </Step>
</Steps>

## Your First Service Request

Now that Servineo is running, let's create your first service request:

<Steps>
  <Step title="Sign up as a requester">
    1. Click "Sign Up" in the navigation menu
    2. Choose to register as a **Requester** (someone who needs services)
    3. Fill in your details or use Google OAuth
    4. Accept the terms and conditions

    The signup flow is located in `src/app/[locale]/signUp/`
  </Step>

  <Step title="Browse services">
    After logging in, browse available services:
    - Plumbing (Gasfitería)
    - Electrical work (Electricistas)
    - Carpentry (Carpintería)
    - Cleaning (Limpieza)
    - And many more...

    Service images are available in the `public/` directory.
  </Step>

  <Step title="Create a job request">
    1. Navigate to "Ask for Help" or click on a service category
    2. Fill out the job request form with:
       - Service type
       - Description of work needed
       - Location
       - Preferred date and time
    3. Submit your request

    The job request form is at `src/app/[locale]/ask-for-help/`
  </Step>

  <Step title="View job offers">
    Once your request is posted, fixers can submit offers. View them at:
    - Navigate to "Job Offers" in your dashboard
    - Review fixer profiles, ratings, and pricing
    - Accept an offer to proceed

    Job offers are managed in `src/app/[locale]/job-offer-list/`
  </Step>
</Steps>

## Understanding the User State

Servineo uses Redux Toolkit for state management. Access the current user throughout the app:

```tsx
import { useSelector } from 'react-redux';
import { IUser } from '@/types/user';

interface RootState {
  user: {
    user: IUser | null;
    loading: boolean;
    isAuthenticated: boolean;
  };
}

function MyComponent() {
  const { user, loading } = useSelector((state: RootState) => state.user);
  
  if (loading) return <div>Loading...</div>;
  
  const isFixer = user?.role === 'fixer';
  const isRequester = user?.role === 'requester';
  
  return (
    <div>
      <h1>Welcome, {user?.name}</h1>
      <p>Role: {user?.role}</p>
    </div>
  );
}
```

The Redux store is configured in `src/app/redux/store.ts:12`

## Multi-language Support

Servineo supports English and Spanish out of the box:

- **English**: `http://localhost:3000/en`
- **Spanish**: `http://localhost:3000/es`

Translations are managed with `next-intl` and stored in `messages/en.json` and `messages/es.json`.

```tsx
import { useTranslations } from 'next-intl';

function MyComponent() {
  const t = useTranslations('home');
  
  return <h1>{t('title')}</h1>;
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 is already in use">
    If port 3000 is occupied, you can specify a different port:
    
    ```bash
    npm run dev -- --port 3001
    ```
    
    Or modify the script in `package.json:6`
  </Accordion>

  <Accordion title="API connection errors">
    Ensure your backend API is running on port 8000. The Next.js config proxies `/api` requests to `http://127.0.0.1:8000/api` (see `next.config.ts:82`).
    
    If your backend runs on a different port, update the rewrite configuration.
  </Accordion>

  <Accordion title="Module not found errors">
    Clear your node_modules and reinstall:
    
    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="Environment variables not loading">
    - Ensure `.env.local` is in the root directory
    - Restart the dev server after changing environment variables
    - Variables must be prefixed with `NEXT_PUBLIC_` to be accessible in browser
  </Accordion>
</AccordionGroup>
