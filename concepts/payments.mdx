Servineo's payment system handles transactions between requesters and fixers, supporting multiple payment methods including cash, QR codes, and credit cards.

## Payment Status Flow

Payments progress through several states defined in `src/utils/types.ts`:

```typescript
export type PaymentStatus = 'pending' | 'paid' | 'failed';
```

<Steps>
  <Step title="pending">
    Payment has been initiated but not yet confirmed
  </Step>
  
  <Step title="paid">
    Payment successfully processed and confirmed
  </Step>
  
  <Step title="failed">
    Payment failed or was rejected
  </Step>
</Steps>

## Payment Summary

The core payment summary interface is defined in `src/utils/types.ts`:

```typescript
export interface PaymentSummary {
  id: string;
  code: string | null;        // Payment verification code
  status: PaymentStatus;
  amount: {
    total: number;
    currency: string;          // 'BOB' or 'USD'
  };
  codeExpired?: boolean;       // Whether payment code has expired
  codeExpiresAt?: string;      // Expiration timestamp
  createdAt?: string;
}
```

## Creating Payments

The payment service is defined in `src/app/[locale]/payment/service/payment.ts`:

```typescript
import { createCashPayment } from '@/app/[locale]/payment/service/payment';

const payment = await createCashPayment({
  jobId: 'job-123',
  requesterId: 'requester-456',
  fixerId: 'fixer-789',
  subTotal: 150,
  service_fee: 15,          // 10% platform fee
  discount: 0,
  currency: 'BOB',
  paymentMethods: 'cash',
  commissionRate: 0.1       // 10% commission
});
```

## Confirming Payments

For cash and QR payments, confirmation requires a verification code:

```typescript
import { confirmPayment } from '@/app/[locale]/payment/service/payment';

try {
  const result = await confirmPayment('payment-id-123', 'ABC123');
  
  console.log(result);
  // {
  //   message: 'Payment confirmed successfully',
  //   data: {
  //     id: 'payment-id-123',
  //     total: 165,
  //     status: 'paid',
  //     paidAt: '2024-03-15T14:25:00Z'
  //   }
  // }
} catch (error) {
  console.error('Payment confirmation failed:', error.message);
}
```

<Warning>
  Payment codes expire after a set time period. Check `codeExpired` and `codeExpiresAt` before attempting confirmation.
</Warning>

## Payment Calculation

Here's how to calculate the total payment:

```typescript
function calculateTotal(
  subtotal: number,
  commissionRate: number = 0.1,
  discount: number = 0
): number {
  const serviceFee = subtotal * commissionRate;
  const total = subtotal + serviceFee - discount;
  return Math.round(total * 100) / 100; // Round to 2 decimals
}

const subtotal = 150;
const total = calculateTotal(subtotal);
console.log(total); // 165 (150 + 15 service fee)
```

## Integration with Jobs

Payments are tightly integrated with the job lifecycle:

```typescript
import { JobRequest } from '@/types/job-request';

// Job status progression
'pending' → 'accepted' → 'in_progress' → 'completed' → 'paid'
//                                                         ↑
//                                                Payment confirms here
```

Once a job is marked as `paid`, the transaction is complete and funds are released to the fixer's wallet.
