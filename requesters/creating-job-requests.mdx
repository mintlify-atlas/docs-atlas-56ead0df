## Overview

Job requests allow you to submit your specific service needs directly to fixers. Unlike browsing job offers, requests let you describe exactly what you need and receive custom proposals.

## Opening the Job Request Form

You can create a job request from:

1. **Job Offer Modal** - Click "Solicitar trabajo" button when viewing a fixer's offer
2. **Job Request Page** - Navigate to `/job-request`

<Note>
  You must be logged in as a requester to submit job requests. Fixers cannot request jobs.
</Note>

```tsx Components/Job-offers/Job-offer-modal.tsx
const handleRequestJobClick = () => {
  if (!isAuthenticated) {
    router.push('/signUp');
    return;
  }
  if (user?.role === 'fixer') {
    return;
  }
  setIsMapModalOpen(true);
};
```

## Job Request Form

The job request modal (`JobRequestModal`) contains a comprehensive form with validation.

### Required Fields

<Steps>
  <Step title="Job Motive">
    Enter the main reason for the service request.
    
    **Validation Rules:**
    - Required field
    - Text only (numbers are automatically removed)
    - Must be completed before description field is enabled
    
    ```tsx app/[locale]/job-request/ModalFormMap/JobRequestForm.tsx
    const validateTextOnly = (text: string): string => {
      return text.replace(/[0-9]/g, '');
    };
    ```
  </Step>
  
  <Step title="Job Description">
    Provide detailed information about the work to be performed.
    
    **Features:**
    - Disabled until job motive is filled
    - Text-only input (auto-validates)
    - Multi-line textarea (4 rows)
    - Required field
    
    ```tsx
    <textarea
      id='jobDescription'
      name='jobDescription'
      value={formData.jobDescription}
      onChange={handleInputChange}
      disabled={loading || !formData.jobMotive.trim()}
      required
      rows={4}
    />
    ```
  </Step>
  
  <Step title="Time Availability">
    Specify when you're available for the service.
    
    **Time Constraints:**
    - Start time: 7:00 AM - 9:00 PM
    - End time: 7:00 AM - 9:00 PM
    - Minimum duration: 2 hours
    - Maximum duration: 8 hours
    - End time must be after start time
    
    ```tsx app/[locale]/job-request/ModalFormMap/JobRequestForm.tsx
    const validateTimes = (start: string, end: string): string => {
      const startTime = new Date(`2000-01-01T${start}`);
      const endTime = new Date(`2000-01-01T${end}`);
      
      const minTime = new Date(`2000-01-01T07:00`);
      const maxTime = new Date(`2000-01-01T21:00`);
      
      if (startTime < minTime || startTime > maxTime) {
        return 'La hora de inicio debe estar entre 7:00 y 21:00';
      }
      
      const durationMs = endTime.getTime() - startTime.getTime();
      const durationHours = durationMs / (1000 * 60 * 60);
      
      if (durationHours < 2) {
        return 'La duración mínima del trabajo es de 2 horas';
      }
      
      if (durationHours > 8) {
        return 'La duración máxima del trabajo es de 8 horas';
      }
      
      return '';
    };
    ```
  </Step>
</Steps>

### Interactive Map

The map component (`MapJobRequest`) provides:

- **Visual Location Selection** - Drag marker to exact position
- **Enabled/Disabled States** - Only active when "modify" is selected
- **Initial Position** - Defaults to saved location or La Paz coordinates
- **Position Callback** - Updates form data when marker moves

```tsx app/[locale]/job-request/ModalFormMap/JobRequestForm.tsx
<MapJobRequestMemo

  onPositionChange={handlePositionChange}
```

## Submitting the Request

### Submit Button State

The submit button is only enabled when:

```tsx app/[locale]/job-request/ModalFormMap/JobRequestForm.tsx
const isFormValid = useMemo(() => {
  return (
    formData.jobMotive.trim() !== '' &&
    formData.jobDescription.trim() !== '' &&
    formData.startTime !== '' &&
    formData.endTime !== '' &&
    !timeError
  );
}, [
  formData.jobMotive,
  formData.jobDescription,
  formData.startTime,
  formData.endTime,
  timeError,
]);
```

### On Submit

When you click "Guardar":

1. **Final Validation** - All fields checked one last time
2. **Location Processed** - New location or saved location determined
3. **Data Submission** - Request sent to backend with fixer ID and offer ID
4. **Modal Closes** - Success state triggers modal dismissal

```tsx
const handleSubmit = (e: React.FormEvent) => {
  e.preventDefault();
  
  if (!formData.jobMotive.trim()) {
    setFieldErrors((prev) => ({ ...prev, jobMotive: 'Este campo es requerido' }));
    return;
  }
  
  onSubmit(formData, newLocation);
};
```

## Form Data Structure

The submitted job request contains:

```typescript types/job-request.ts
interface JobRequestData {
  jobMotive: string;              // Why you need the service
  jobDescription: string;         // Detailed work description
  locationOption: 'keep' | 'modify';  // Location choice
  startTime: string;              // Start time (HH:MM format)
  endTime: string;                // End time (HH:MM format)
  suggestedRate?: string;         // Optional budget
}

interface Location {
  lat: number;                    // Latitude
  lng: number;                    // Longitude
}
```

<Warning>
  Double-check all information before submitting. Job requests are sent directly to fixers and cannot be edited after submission.
</Warning>

## After Submission

Once submitted:

1. **Fixer Notification** - The fixer receives your job request
2. **Offer Creation** - Fixer can create a custom offer for you
3. **Notification** - You'll be notified when offers arrive
4. **Review Offers** - Check [Job Offers](/requesters/job-offers) to review and accept
