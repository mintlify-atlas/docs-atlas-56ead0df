## System Requirements

### Required Software

- **Node.js**: Version 20.x or higher
- **npm**: Version 11.x or higher (included with Node.js)
- **Git**: For version control
- **Code Editor**: VS Code recommended with TypeScript support

### Recommended System Specs

- **RAM**: 8GB minimum, 16GB recommended
- **Disk Space**: 2GB for dependencies and build artifacts
- **OS**: Windows 10+, macOS 10.15+, or Linux (Ubuntu 20.04+)

## Installation Steps

<Steps>
  <Step title="Verify Node.js installation">
    Check that you have the correct Node.js version:

    ```bash
    node --version
    # Should output v20.x.x or higher

    npm --version
    # Should output 11.x.x or higher
    ```

    If you need to install or update Node.js, download it from [nodejs.org](https://nodejs.org/)

    <Tip>
      Consider using [nvm](https://github.com/nvm-sh/nvm) (Node Version Manager) to manage multiple Node.js versions.
    </Tip>
  </Step>

  <Step title="Clone the repository">
    Clone the Servineo frontend repository:

    ```bash
    git clone <repository-url>
    cd servineo-frontend
    ```

    Verify the project structure:

    ```bash
    ls -la
    # You should see: src/, public/, package.json, next.config.ts, etc.
    ```
  </Step>

  <Step title="Install dependencies">
    Install all npm packages:

    ```bash
    npm install
    ```

    This installs 66 dependencies and 91 dev dependencies defined in `package.json`.

    <Warning>
      The installation may take 2-5 minutes depending on your internet connection and system performance.
    </Warning>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env.local` file in the project root:

    ```bash
    touch .env.local
    ```

    Add the required environment variables (see [Environment Variables](#environment-variables) section below).
  </Step>

  <Step title="Verify installation">
    Start the development server to verify everything is set up correctly:

    ```bash
    npm run dev
    ```

    If successful, you'll see:
    ```
    ▲ Next.js 15.5.4
    - Local:        http://localhost:3000
    - Ready in 2.5s
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser.
  </Step>
</Steps>

## Environment Variables

Create a `.env.local` file with the following configuration:

```bash .env.local
# ============================================
# API Configuration
# ============================================
# Base URL for the Next.js app
NEXT_PUBLIC_API_URL=http://localhost:3000

# Backend API server (runs on port 8000)
NEXT_PUBLIC_BACKEND=http://127.0.0.1:8000
NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000/api

# ============================================
# Authentication (Google OAuth)
# ============================================
NEXT_PUBLIC_GOOGLE_CLIENT_ID=your_google_client_id_here

# Get your Google Client ID from:
# https://console.cloud.google.com/apis/credentials

# ============================================
# Security (reCAPTCHA)
# ============================================
NEXT_PUBLIC_RECAPTCHA_SITE_KEY=your_recaptcha_site_key_here

# Get your reCAPTCHA keys from:
# https://www.google.com/recaptcha/admin

# ============================================
# Payments (Stripe) - Optional
# ============================================
# NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_...

# ============================================
# Environment
# ============================================
NODE_ENV=development
```

<Warning>
  **Important Security Notes:**
  - Never commit `.env.local` to version control
  - Only variables prefixed with `NEXT_PUBLIC_` are exposed to the browser
  - Keep sensitive keys (API secrets, private keys) on the server side only
  - Use different keys for development and production environments
</Warning>

### API Proxy Configuration

Servineo uses Next.js rewrites to proxy API requests to the backend. This is configured in `next.config.ts:82`:

```typescript next.config.ts
async rewrites() {
  return [
    {
      source: '/api/:path*',
      destination: 'http://127.0.0.1:8000/api/:path*',
    },
  ];
}
```

This means:
- Frontend requests to `/api/users` are proxied to `http://127.0.0.1:8000/api/users`
- Avoids CORS issues during development
- Keeps the backend URL hidden from the client

## Redux Store Configuration

Servineo uses Redux Toolkit for state management. The store is configured in `src/app/redux/store.ts`:

```typescript src/app/redux/store.ts
import { configureStore } from '@reduxjs/toolkit';
import { setupListeners } from '@reduxjs/toolkit/query';
import { baseApi } from './services/baseApi';
import userReducer from './slice/userSlice';
import fixerReducer from './slice/fixerSlice';
import filterReducer from './slice/filterSlice';
import jobOffersReducer from './slice/jobOffersSlice';
import jobOfertReducer from './slice/jobOfert';
import fixersByJobReducer from './slice/fixersByJobSlice';
import logger from 'redux-logger';

export const store = configureStore({
  reducer: {
    [baseApi.reducerPath]: baseApi.reducer,
    user: userReducer,
    fixer: fixerReducer,
    filters: filterReducer,
    jobOffers: jobOffersReducer,
    jobOfert: jobOfertReducer,
    fixersByJob: fixersByJobReducer,
  },
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware()
      .concat(baseApi.middleware)
      .concat(logger),
  devTools: process.env.NODE_ENV !== 'production',
});

setupListeners(store.dispatch);

export type RootState = ReturnType<typeof store.getState>;
export type AppDispatch = typeof store.dispatch;
```

### Store Slices

- **user**: Current user authentication and profile data
- **fixer**: Fixer-specific data and profiles
- **filters**: Search and filter state
- **jobOffers**: Job offers from fixers
- **jobOfert**: Individual job request details
- **fixersByJob**: Fixers associated with specific jobs
- **baseApi**: RTK Query API endpoints

## Common Issues

<AccordionGroup>
  <Accordion title="npm install fails with permission errors">
    On Unix systems, you might need to fix npm permissions:
    
    ```bash
    sudo chown -R $USER:$(id -gn $USER) ~/.npm
    ```
    
    Or use `nvm` to avoid sudo altogether.
  </Accordion>

  <Accordion title="Turbopack errors during build">
    If you encounter Turbopack issues, try:
    
    ```bash
    rm -rf .next
    npm run dev
    ```
    
    Turbopack cache is stored in `.next/` and can sometimes become corrupted.
  </Accordion>

  <Accordion title="TypeScript errors in IDE but build succeeds">
    The current configuration ignores TypeScript errors during builds. To fix:
    
    1. Run `npx tsc --noEmit` to see all type errors
    2. Fix errors incrementally
    3. Remove `ignoreBuildErrors: true` from `next.config.ts:14`
  </Accordion>

  <Accordion title="Images not loading from Google Drive">
    Ensure image remote patterns are configured in `next.config.ts:24`:
    
    ```typescript
    images: {
      remotePatterns: [
        { hostname: 'lh3.googleusercontent.com' },
        { hostname: 'drive.google.com' },
      ],
    }
    ```
  </Accordion>

  <Accordion title="Redux DevTools not working">
    Redux DevTools only work in development mode. Ensure:
    
    ```bash
    NODE_ENV=development npm run dev
    ```
    
    Install the [Redux DevTools Extension](https://github.com/reduxjs/redux-devtools) for your browser.
  </Accordion>
</AccordionGroup>
