The Servineo user management system provides administrators with tools to monitor active appointments, track user activity, and manage user roles across the platform.

## Overview

The user management interface is accessible at `/tracking-appointments` and provides real-time tracking of appointments, fixer statistics, and geographic distribution of services.

## Appointment Tracking

### Tracking API Integration

The system uses Redux Toolkit Query to fetch appointment data:

```typescript

  useGetMapLocationsQuery,
  useGetTrackingMetricsQuery,
  useGetFixerStatsQuery,
```

### Appointment Data Structure

```typescript
interface ApiAppointment {
  _id: string;
  lat: string | number;
  lon: string | number;
  fixerName?: string;
  requesterName?: string;
  current_requester_name?: string;
  date?: string;
  starting_time?: string;
  status?: string;
  schedule_state?: string;
}

interface MappedAppointment {
  id: string;
  fixerName: string;
  requesterName: string;
  date: string;
  status: string;
  lat: number;
  lng: number;
  service: string;
}
```

### Data Transformation

Appointments are transformed and filtered for display:

```typescript
const filteredAppointments = useMemo(() => {
  if (!rawMapData) return [];

  return (rawMapData as ApiAppointment[])
    .map((app): MappedAppointment => ({
      id: app._id,
      fixerName: app.fixerName || t('map.unknown'),
      requesterName: app.requesterName || app.current_requester_name || t('map.client'),
      date: app.date || app.starting_time || '',
      status: app.status || app.schedule_state || 'unknown',
      lat: Number(app.lat),
      lng: Number(app.lon),
      service: '',
    }))
    .filter((app) => !isNaN(app.lat) && !isNaN(app.lng))
    .filter((app) => {
      if (!startDate || !endDate) return true;
      const appointmentDate = new Date(app.date);
      const start = new Date(startDate);
      const end = new Date(endDate);
      end.setHours(23, 59, 59);
      return appointmentDate >= start && appointmentDate <= end;
    });
}, [rawMapData, startDate, endDate, t]);
```

## User Roles

### Role Types

The system tracks three primary user roles:

1. **Visitor**: Unauthenticated users browsing the platform
2. **Requester**: Users requesting services
3. **Fixer**: Service providers
4. **Admin**: System administrators

### Role-Based Activity

Activity logs track role-specific actions:

```typescript
interface JobLog {
  userId: string;
  role: 'visitor' | 'requester' | 'fixer';
  type: 'login' | 'search' | 'click' | 'review' | 'session_start' | 'session_end';
  metadata: JobStatusMetadata;
  timestamp: string;
}
```
