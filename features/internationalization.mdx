## Overview

Servineo supports multiple languages using `next-intl`, providing a seamless bilingual experience for English and Spanish speakers. The i18n system covers all user-facing content including UI labels, validation messages, and dynamic content.

## Configuration

### Routing Setup

Language routing is configured in `src/i18n/routing.ts`:

```tsx title="src/i18n/routing.ts"
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  // A list of all locales that are supported
  locales: ['en', 'es'],
  // Used when no locale matches
  defaultLocale: 'en',
});
```

### Middleware

Automatic locale detection and routing via middleware:

```tsx title="src/middleware.ts"
import createMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';

export default createMiddleware({
  locales: routing.locales,
  defaultLocale: routing.defaultLocale,
  localeDetection: true,
  localeCookie: true,
});

export const config = {
  matcher: ['/((?!api|_next/static|_next/image|favicon.ico).*)'],
};
```

**Features:**

- **Automatic locale detection** from browser settings
- **Locale cookie** to persist user preference
- **Path-based routing** (e.g., `/en/job-offers`, `/es/ofertas`)
- **Static file exclusion** for optimal performance

## Using Translations in Components

### Basic Usage

Import and use the `useTranslations` hook:

```tsx title="Component with translations" {1,4,7-8}
import { useTranslations } from 'next-intl';

export function FiltersPanel() {
  const t = useTranslations('filtersPanel');

  return (
    <h2>{t('filters')}</h2>
    <h3>{t('fixerName')}</h3>
    <h3>{t('city')}</h3>
    <h3>{t('jobCategory')}</h3>
    <button>{t('resetButton.desktop')}</button>
  );
}
```

### Nested Keys

Access nested translation keys with dot notation:

```tsx
const t = useTranslations('advancedSearch');

<h1>{t('pageTitle')}</h1>
<p>{t('resultsCounter.loading')}</p>
<span>{t('fixerName.ranges.ac')}</span>
```

### Dynamic Content

Use variables in translations:

```json
{
  "pagination": {
    "showingResults": "Showing {start} - {end} of {total} results"
  }
}
```

```tsx
const t = useTranslations('pagination');

t('showingResults', { start: 1, end: 10, total: 100 })
// English: "Showing 1 - 10 of 100 results"
// Spanish: "Mostrando 1 - 10 de 100 resultados"
```

### Pluralization

Handle singular/plural forms:

```json
{
  "jobOffers": {
    "map": {
      "availableOffers": {
        "singular": "available offer",
        "plural": "available offers"
      }
    }
  }
}
```

```tsx
const t = useTranslations('jobOffers.map');
const count = offersInRadius;

{count} {count === 1 
  ? t('availableOffers.singular') 
  : t('availableOffers.plural')
}
```

## Language Detection

### From URL Path

Detect current language from the URL:

```tsx title="Detect language from pathname" {1-5}
const currentLanguage = React.useMemo(() => {
  const pathSegments = (pathname || '').split('/').filter(Boolean);
  const langSegment = pathSegments[0];
  return ['en', 'es'].includes(langSegment) ? langSegment : 'es';
}, [pathname]);
```

### Search Suggestions by Language

Pass detected language to backend API:

```tsx title="Language-aware search suggestions" {6}
const { suggestions } = useSearchSuggestions(value, {
  enabled: value.trim().length > 0,
  minLength: 1,
  debounceMs: 300,
  maxResults: 6,
  language: currentLanguage,
});
```

## Best Practices

1. **Use semantic keys** - Organize by feature/component
2. **Avoid hardcoded strings** - Always use translation keys
3. **Keep translations in sync** - Ensure all locales have matching keys
4. **Use descriptive key names** - Make purpose clear
5. **Support pluralization** - Provide singular/plural variants
6. **Include context** - Add comments for translators when needed
7. **Handle missing translations** - Provide fallbacks
8. **Test both languages** - Verify layout works for all locales
9. **Use variables** - For dynamic content insertion
10. **Translate accessibility** - Include ARIA labels and alt text

## Language Switcher

Implement a language switcher component:

```tsx title="Simple language switcher"
import { useRouter, usePathname } from 'next/navigation';

export function LanguageSwitcher() {
  const router = useRouter();
  const pathname = usePathname();

  const switchLanguage = (locale: 'en' | 'es') => {
    const segments = pathname.split('/');
    segments[1] = locale;
    router.push(segments.join('/'));
  };

  return (
    <div>
      <button onClick={() => switchLanguage('en')}>English</button>
      <button onClick={() => switchLanguage('es')}>Español</button>
    </div>
  );
}
```

## Performance Tips

1. **Lazy load translations** - Only load needed namespaces
2. **Cache translations** - Use built-in next-intl caching
3. **Avoid large files** - Split translations by route/feature
4. **Preload common strings** - Load frequently used translations early
