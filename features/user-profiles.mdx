## Overview

Servineo supports two primary user types with distinct profile features: **Requesters** (clients who request services) and **Fixers** (service providers). Each profile type has specific fields and capabilities tailored to their role in the platform.

## User Types

### Requester Profiles

Requesters are clients who book services. Their profiles include:

- **Basic Information**: Name, email, photo
- **Contact**: Phone number
- **Location**: Address with coordinates (lat/lng)
- **Payment Methods**: Preferred payment options
- **Terms Acceptance**: Agreement to platform terms

### Fixer Profiles

Fixers are service providers with additional professional information:

- **All Requester Fields** plus:
- **Services Offered**: Array of service categories (`servicios`)
- **Professional Experience**: Description of work history
- **Work Location**: Service area with coordinates
- **Vehicle Information**: Type of vehicle (if applicable)
- **CI (Identification)**: Identity verification
- **Ratings & Reviews**: Client feedback and ratings

## Profile Setup During Registration

New users complete their profile through a multi-step registration flow:

### Step 1: Phone Number

Users provide their contact number for verification:

```typescript
export async function enviarTelefono(telefono: string): Promise<TelefonoResponse> {
  const token = localStorage.getItem('servineo_token');
  try {
    const res = await fetch(`${BASE_URL}/registrar/telefono`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        Authorization: `Bearer ${token}`,
      },
      body: JSON.stringify({ telefono }),
    });

    const data = await res.json();

    if (!res.ok) {
      throw new Error(`Error ${res.status}: ${data.error || res.statusText}`);
    }

    return data;
  } catch (error) {
    console.error('Error al enviar el teléfono al backend:', error);
    throw error;
  }
}
```

```typescript
export async function enviarUbicacion(
  lat: number,
  lng: number,
  direccion: string | null,
  departamento: string | null,
  pais: string | null,
): Promise<UbicacionResponse> {
  const token = localStorage.getItem('servineo_token');

    return await res.json();
  } catch (error) {
    console.error('Error al enviar la ubicación al backend:', error);
    throw error;
  }
}
```

## Profile Authentication

Profile data is accessed using the stored authentication token:

```typescript
const token = localStorage.getItem('servineo_token');

const headers = {
  'Content-Type': 'application/json',
  Authorization: `Bearer ${token}`,
};
```

This ensures that:
- Only authenticated users can view certain profile details
- Users can only edit their own profiles
- Sensitive information is protected

## Profile Data Storage

### Backend Storage

## Best Practices

1. **Always validate user input** during profile updates
2. **Use optimistic UI updates** for better user experience
3. **Cache profile data** to reduce API calls
4. **Handle missing profile photos** gracefully with default avatars
5. **Validate location coordinates** before saving
6. **Require authentication** for viewing sensitive profile information
7. **Implement proper error handling** for profile updates
8. **Show loading states** while fetching profile data
9. **Use responsive design** for mobile-friendly profiles
10. **Optimize images** for faster loading
