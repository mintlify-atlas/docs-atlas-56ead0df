The Servineo admin dashboard provides a centralized interface for administrators to monitor user statistics, manage system analytics, and access specialized administrative modules.

## Overview

The admin dashboard is accessible at `/user-admin/dashboard` and requires administrator authentication. It displays real-time user statistics, role distribution, session analytics, and provides links to external administrative tools.

## Authentication

### Login Methods

The admin system supports two authentication methods:

1. **Email/Password Authentication**
2. **Google OAuth Integration**

```typescript
// Email/Password Login
const data = await adminAPI.login({ email, password });

if (data.success) {
  localStorage.setItem('adminToken', data.token);
  localStorage.setItem('adminUser', JSON.stringify(data.user));
  router.push(`/${locale}/user-admin/dashboard`);
}
```

```typescript
// Google OAuth Login
const data = await adminAPI.loginWithGoogle(credential);

if (data.success) {
  localStorage.setItem('adminToken', data.token);
  localStorage.setItem('adminUser', JSON.stringify(data.user));
  router.push(`/${locale}/user-admin/dashboard`);
}
```

### Token Verification

The dashboard automatically verifies the admin token on load:

```typescript
const authCheck = await adminAPI.verifyToken(token);

if (!authCheck.valid) {
  localStorage.removeItem('adminToken');
  localStorage.removeItem('adminUser');
  router.push('/es/user-admin');
}
```

### User Role Distribution

The dashboard displays user distribution across four roles:

- **Requesters**: Users who request services
- **Fixers**: Service providers
- **Visitors**: Unregistered users
- **Admins**: System administrators

Each role card shows:
- Total count
- Percentage of total users
- Visual indicator with role-specific styling

```tsx
<div className={`${styles.roleCard} ${styles.requesterCard}`}>
  <div className={styles.roleHeader}>
    <span className={styles.roleIcon}>👤</span>
    <span className={styles.roleName}>Requesters</span>
  </div>
  <div className={styles.roleCount}>
    {userStats?.usersByRole?.requester || 0}
  </div>
  <div className={styles.rolePercentage}>
    {Math.round(((userStats.usersByRole?.requester || 0) / userStats.totalUsers) * 100)}%
  </div>
</div>
```

### Session Start/End Tracking

Two charts track user sessions:

1. **Session Start Chart**: Monitors when users begin sessions
2. **Session End Chart**: Tracks when users close sessions

```typescript
const [startData, endData] = await Promise.all([
  adminAPI.getSessionStartChart(token, dateRange.start, dateRange.end),
  adminAPI.getSessionEndChart(token, dateRange.start, dateRange.end),
```

## Dashboard Features

### Auto-Refresh

Manual refresh capability for user statistics:

```typescript
const handleRefreshStats = async () => {
  setStatsLoading(true);
  try {
    const statsResponse = await adminAPI.getUserStats(token);
    if (statsResponse.success && statsResponse.data) {
      setUserStats(statsResponse.data);
      setLastUpdated(new Date(statsResponse.data.timestamp).toLocaleTimeString());
    }
  } finally {
    setStatsLoading(false);
  }
};
```

### Logout Functionality

```typescript
const handleLogout = () => {
  localStorage.removeItem('adminToken');
  localStorage.removeItem('adminUser');
  router.push('/es/user-admin');
};
```
